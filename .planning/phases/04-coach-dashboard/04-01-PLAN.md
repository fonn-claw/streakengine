Create src/lib/queries/coach.ts with two exported functions and their TypeScript interfaces:

1. **CoachUserCard interface** -- export with fields:
   - userId: number, displayName: string, currentStreak: number, weeklyXp: number
   - lastActiveDate: string | null, thisWeekLogs: number, lastWeekLogs: number
   - engagementRatio: number, riskLevel: "green" | "yellow" | "red", nudgedToday: boolean

2. **getCoachDashboardData(coachId: number): Promise<CoachUserCard[]>** --
   - Use getTodayForUser("UTC") for the coach's date perspective
   - Compute thisWeek = getWeekBounds(today), lastWeek = getWeekBounds(7 days before today)
   - Single Drizzle query: SELECT from users LEFT JOIN streaks ON streaks.userId = users.id WHERE users.role != 'coach'
   - Use SQL subqueries (via sql template tag) for: thisWeekLogs (COUNT habit_logs where date between thisWeek.start and thisWeek.end), lastWeekLogs (same for lastWeek bounds), weeklyXp (SUM xp_earned for this week), nudgedToday (EXISTS nudges where sender_id = coachId and recipient_id = users.id and date = today)
   - In JS post-processing: calculate engagementRatio = lastWeekLogs === 0 ? 1 : thisWeekLogs / lastWeekLogs
   - Risk classification: ratio < 0.7 = "red", ratio < 0.9 = "yellow", else "green"
   - Also check: if user createdAt is less than 14 days ago, force "green" regardless of ratio
   - Sort result: red first, then yellow, then green. Within same risk level, sort by engagementRatio ascending (worst first)
   - COALESCE all numeric subqueries to 0 to handle nulls

3. **HabitDay interface** -- export with fields:
   - habitName: string, habitIcon: string | null, days: { date: string; completed: boolean }[]

4. **getUserHabitHeatmap(userId: number): Promise<HabitDay[]>** --
   - Get today via getTodayForUser("UTC"), compute weekBounds = getWeekBounds(today)
   - Query all habits from the habits table
   - Query habitLogs for this userId WHERE date BETWEEN weekBounds.start and weekBounds.end
   - Generate array of 7 dates (Mon through Sun of this week)
   - For each habit, build a HabitDay with days array: for each date, completed = true if a log exists for that habit+date combo with progress >= targetCount (or any log exists for targetCount=1 habits)
   - Return array sorted by habit name

Import from "@/db" for db, from "@/db/schema" for tables, from "drizzle-orm" for sql/eq/ne/and/between, from "@/lib/utils" for date helpers. Follow the same import style as src/lib/queries/profile.ts.
