Create 3 files with pure domain logic (zero DB imports):

**src/lib/engine/xp.ts** — XP and leveling calculations:
- `calculateXP(baseXP: number, multiplier: number): number` — Returns `Math.round(baseXP * multiplier)`. Applies streak multiplier to base habit XP.
- `xpForLevel(level: number): number` — Cumulative XP threshold for reaching a level. Formula: `sum(i=1 to level) of (i * 100 + (i - 1) * 50)`. Expected values: Level 1 = 100, Level 2 = 350, Level 3 = 750, Level 4 = 1300, Level 5 = 2000.
- `getLevelFromXP(totalXP: number): number` — Returns the highest level where `xpForLevel(level) <= totalXP`. A user with 0 XP is level 0 (not level 1). A user with 100 XP is level 1. A user with 349 XP is still level 1. A user with 350 XP is level 2.
- `getLevelProgress(totalXP: number): { level: number; currentLevelXP: number; nextLevelXP: number; progressXP: number }` — Returns current level, the XP threshold of current level, the XP threshold of next level, and how much XP into the current level the user is. Used to render the XP progress bar.

**src/lib/engine/streak.ts** — Streak evaluation and multiplier:
- `evaluateStreak(params: { currentStreak: number; lastActiveDate: string | null; today: string; yesterday: string; freezeAvailable: boolean; freezesUsedThisWeek: number; multiplier: number; longestStreak: number; streakStartDate: string | null; }): StreakEvaluation` — Pure function that takes current streak state + today/yesterday date strings, returns a StreakEvaluation object describing what should change. Logic:
  - If `lastActiveDate === today`: no change (already active today), return `{ action: "already_active" }`
  - If `lastActiveDate === yesterday`: streak continues, return `{ action: "increment", newStreak: currentStreak + 1, newLongest: Math.max(longestStreak, currentStreak + 1) }`
  - If `lastActiveDate` is older than yesterday AND `freezeAvailable` AND `freezesUsedThisWeek < 1`: freeze saves streak, return `{ action: "freeze_used", freezeMessage: "Freeze saved your streak!" }`
  - If `lastActiveDate` is older than yesterday AND no freeze available: streak breaks, return `{ action: "break", newStreak: 1, freezeMessage: null }`
  - If `lastActiveDate` is null (first ever): return `{ action: "first_ever", newStreak: 1 }`
- Define `StreakEvaluation` type as a discriminated union on `action` field. Each variant carries only the fields relevant to that action.
- `calculateMultiplier(consecutiveCompleteWeeks: number): number` — Returns multiplier: 0 weeks = 1.0, 1 week = 1.0, 2 weeks = 1.2, 3 weeks = 1.3, 4+ weeks = 1.5.
- `isWeekComplete(habitLogDates: string[], weekStart: string, weekEnd: string): boolean` — Checks if every day in the Mon-Sun range has at least one habit log. Used for multiplier recalculation.

**src/lib/utils.ts** — Timezone-safe date helpers:
- `getTodayForUser(timezone: string): string` — Uses `new Intl.DateTimeFormat("en-CA", { timeZone: timezone, year: "numeric", month: "2-digit", day: "2-digit" }).format(new Date())` to produce "YYYY-MM-DD" string in user's timezone. "en-CA" locale is the key — it outputs ISO date format.
- `getYesterdayForUser(timezone: string): string` — Same as above but for yesterday. Create a Date, subtract 1 day via `setDate(getDate() - 1)`, format.
- `getWeekBounds(date: string): { start: string; end: string }` — Given a date string, returns the Monday and Sunday of that week in "YYYY-MM-DD" format. Use date-fns `startOfWeek` with `{ weekStartsOn: 1 }` (Monday) and `endOfWeek`.

Export all functions and types.

**src/lib/queries/habits.ts** — Fetch today's quests:
- `getTodayQuests(userId: number, timezone: string): Promise<Quest[]>` where Quest type is `{ id: number; name: string; icon: string | null; xpReward: number; targetCount: number; progress: number; completed: boolean; xpEarned: number; }`.
- Use a left join: `db.select({ id: habits.id, name: habits.name, icon: habits.icon, xpReward: habits.xpReward, targetCount: habits.targetCount, progress: habitLogs.progress, xpEarned: habitLogs.xpEarned }).from(habits).leftJoin(habitLogs, and(eq(habitLogs.habitId, habits.id), eq(habitLogs.userId, userId), eq(habitLogs.date, today)))`.
- Map results: for each row, `completed` is true when `targetCount === 1 ? progress !== null : (progress ?? 0) >= targetCount`. `progress` defaults to 0 if null.
- Sort: incomplete quests first sorted by xpReward descending, then completed quests below. This is the locked quest card ordering from CONTEXT.md.
- Import `getTodayForUser` from `@/lib/utils`.

**src/lib/queries/streaks.ts** — Fetch streak state with on-load evaluation:
- `getStreakState(userId: number, timezone: string): Promise<StreakState>` where StreakState is `{ currentStreak: number; longestStreak: number; multiplier: number; freezeAvailable: boolean; freezeMessage: string | null; lastActiveDate: string | null; }`.
- Fetch the user's streak record from DB. If none exists, return defaults (streak 0, multiplier 1.0, freeze available).
- Call `evaluateStreak()` from engine with today/yesterday from user's timezone. If the evaluation result indicates a break or freeze, UPDATE the streaks table in the DB and return the updated state. This handles the "on next login, check if previous day had 0 completions" requirement (STRK-03).
- To check if previous day had completions, query `habitLogs` where `userId` and `date = yesterday`. If count > 0, previous day was active (streak evaluation will see lastActiveDate). If count = 0 and lastActiveDate is not yesterday, break/freeze logic applies.
- IMPORTANT: Only evaluate streak gap if `lastActiveDate` is not today and not yesterday. If it IS yesterday, the streak state is fine — it will increment when user completes a habit today.

**src/lib/actions/habits.ts** — Server actions for habit completion:
- Mark file as `"use server"`.
- `completeHabit(habitId: number): Promise<{ xpEarned: number; leveledUp: boolean; newLevel: number; newTotalXp: number; streakIncremented: boolean; newStreak: number; }>` — The main action for single-completion habits (Exercise, Meditation, Sleep):
  1. Call `requireAuth()` to get session.
  2. Fetch user record (need timezone, totalXp, level).
  3. Fetch habit record (need xpReward).
  4. Fetch user's streak record (need multiplier).
  5. Calculate XP: `calculateXP(habit.xpReward, streak.multiplier)`.
  6. Get `previousLevel` from `getLevelFromXP(user.totalXp)`.
  7. Get today's date string via `getTodayForUser(user.timezone)`.
  8. Check if any habitLogs exist for this user + today (to know if this is first completion today for streak).
  9. In a single operation (Neon HTTP does not support multi-statement transactions, so use sequential writes):
     a. Insert into habitLogs: `{ userId, habitId, date: today, progress: 1, xpEarned }`. Use `.onConflictDoNothing()` on the unique index to silently handle double-clicks.
     b. If the insert succeeded (check via returning or catch), update user totalXp: `sql\`total_xp + ${xpEarned}\`` and recalculate level via `getLevelFromXP(newTotalXp)`, update user.level if changed.
     c. If this is the first completion today (no prior logs), call `evaluateStreak()` and update the streaks table accordingly: increment currentStreak, update lastActiveDate to today, update longestStreak if needed.
  10. Compute `newLevel = getLevelFromXP(newTotalXp)`, `leveledUp = newLevel > previousLevel`.
  11. Call `revalidatePath("/")`.
  12. Return `{ xpEarned, leveledUp, newLevel, newTotalXp, streakIncremented, newStreak }`.

- `incrementProgress(habitId: number): Promise<{ progress: number; completed: boolean; xpEarned: number; leveledUp: boolean; newLevel: number; newTotalXp: number; streakIncremented: boolean; newStreak: number; }>` — For partial-progress habits (Water with target_count=8):
  1. Same auth and data fetching as completeHabit.
  2. Check if a habitLog exists for this user + habit + today. If yes, increment its progress by 1. If no, insert with progress = 1.
  3. After increment, check if `newProgress >= habit.targetCount`. If yes, this is the completion moment — award XP (same as completeHabit). If no, just return the new progress without XP.
  4. Streak logic: same as completeHabit — if this is the first log entry today, check streak.
  5. Return `{ progress: newProgress, completed: newProgress >= targetCount, xpEarned, leveledUp, newLevel, newTotalXp, streakIncremented, newStreak }`.

- IMPORTANT: Since Neon HTTP driver does NOT support `db.transaction()`, use sequential queries. The unique index on habit_logs handles race conditions for double-completion. Wrap the insert in try/catch — if it throws a unique constraint error, return the existing state without awarding XP.
